# Claude Code Desktop — Use Cases & Interface Specification

## Overview

Полный прототип десктопного приложения для работы с Claude Code, покрывающий весь цикл взаимодействия: от создания сессии до завершения задачи. Документ описывает каждый юзкейс, реализованный в интерфейсе, с акцентом на отображение процессов инференса.

---

## 1. Управление сессиями

### 1.1 Создание новой сессии
Кнопка «New Session» в сайдбаре. Каждая сессия привязана к рабочей директории проекта (`~/convervox`, `~/api-server` и т.д.) и получает автоматическое название по первому запросу.

### 1.2 Переключение между сессиями
Список сессий в сайдбаре с отображением: название, путь проекта, время последней активности. Активная сессия выделяется фоном. Зелёный индикатор у сессий, в которых Claude сейчас работает.

### 1.3 Состояние соединения
Top bar показывает статус подключения к Claude Code серверу (`connected` с зелёной точкой). Покрывает кейсы: connected, reconnecting, disconnected.

### 1.4 Информация о модели
В сайдбаре отображается текущая версия CLI (`v1.0.42`) и активная модель (`opus-4`). В настройках — полный селектор моделей.

## 2. Ввод сообщений

### 2.1 Текстовый ввод
Textarea с auto-resize (от одной строки до 160px). Поддерживает `⌘ Enter` для отправки и `Shift+Enter` для переноса строки. Подсказка-плейсхолдер: «Ask Claude to code something...».

### 2.2 Голосовой ввод
Кнопка микрофона с двумя состояниями: неактивный (серый) и активный (акцентный цвет с пульсирующей анимацией `mic-pulse`). Нажатие переключает запись — голос транскрибируется в текст инпута.

### 2.3 Прикрепление файлов — кнопка
Кнопка-скрепка открывает системный диалог выбора файлов. Выбранные файлы появляются как pills над инпутом (имя файла + кнопка удаления).

### 2.4 Прикрепление файлов — Drag & Drop
Перетаскивание файлов на область чата активирует полноэкранный оверлей с backdrop-blur, иконкой скрепки и текстом «Drop files to attach». Файлы добавляются при отпускании.

### 2.5 File Pills
Прикреплённые файлы отображаются как компактные pill-элементы: иконка файла, имя, крестик для удаления. Анимация `fade-in` при появлении. Отображаются и в инпуте (до отправки), и в сообщениях пользователя (после отправки).

---

## 3. Режимы работы (Input Toggles)

### 3.1 Auto-approve
Тогл под инпутом. Когда активен — Claude не запрашивает разрешение на операции, выполняет всё автоматически.

### 3.2 Loop mode
Тогл под инпутом (активен по умолчанию). Claude итеративно выполняет задачу: пишет код → запускает тесты → читает ошибки → исправляет → повторяет до успеха.

### 3.3 Verbose
Тогл под инпутом. Включает отображение расширенного thinking — внутренние рассуждения Claude показываются в чате.

## 4. Интерфейс инференса (главный блок)

Это ключевая часть интерфейса — отображение всех этапов работы Claude в реальном времени.

### 4.1 Thinking (размышление)
**Компонент:** `ThinkingBlock` с фазой `thinking`  
**Визуал:** фиолетовая левая граница, иконка мозга, метка «Thinking»  
**Состояния:**
- Активное (без контента) — иконка + метка + пульсирующие точки (`PulsingDots`)
- Завершённое (с контентом) — иконка + метка + курсивный текст размышления

**Кейс:** Claude анализирует запрос пользователя, формирует подход к решению. Пример: *«I need to refactor the voice pipeline from WebSocket to WebRTC...»*

### 4.2 Planning (планирование)
**Компонент:** `ThinkingBlock` с фазой `planning`  
**Визуал:** жёлтая левая граница, иконка слоёв, метка «Planning»  
**Кейс:** Claude составляет пронумерованный план действий. Пример: *«Plan: 1) Create RTCPeerConnection wrapper 2) Replace WS message handlers...»*

### 4.3 Reading files (чтение файлов)
**Компонент:** `ThinkingBlock` с фазой `reading`  
**Визуал:** синяя левая граница, иконка глаза, метка «Reading files»  
**Кейс:** Claude читает файлы из проекта для понимания контекста. Пример: *«Reading voice-pipeline.ts, rtc-config.json...»*

### 4.4 Searching codebase (поиск по кодовой базе)
**Компонент:** `ThinkingBlock` с фазой `searching`  
**Визуал:** синяя левая граница, иконка поиска, метка «Searching codebase»  
**Кейс:** Claude ищет по файлам и символам в проекте (grep, символьный поиск).

### 4.5 Sub-agent (работа субагента)
**Компонент:** `ThinkingBlock` с фазой `subagent`  
**Визуал:** зелёная левая граница, иконка users, метка «Sub-agent working»  
**Кейс:** Claude запускает вложенного агента для параллельной задачи. Пример: *«Running sub-agent to verify test coverage and fix any import issues...»*

### 4.6 Tool Calls (вызовы инструментов)
**Компонент:** `ToolCallBlock`  
**Три состояния:**
- **running** — спиннер (иконка refresh с анимацией `spin`), имя тула, путь файла, пульсирующие точки
- **done** — зелёная галочка, имя тула, путь файла, время выполнения (`0.2s`, `3.4s`)
- **error** — красная иконка ошибки, имя тула

**Поддерживаемые типы тулов:**
- `Read` — чтение файла (с указанием пути)
- `Write` — создание нового файла
- `Edit` — редактирование существующего файла
- `Search` — поиск по кодовой базе
- `Bash` — выполнение команды в терминале

Тул-коллы группируются в стек — последовательность операций визуально считываемая как пайплайн.

### 4.7 Code Blocks (блоки кода)
**Компонент:** `CodeBlock`  
**Элементы:**
- Хедер: иконка кода, имя файла или языка, кнопка копирования
- Тело: моноширинный шрифт (JetBrains Mono), горизонтальный скролл
- Кнопка Copy: иконка clipboard → зелёная галочка с auto-reset через 2 секунды

**Кейс:** Claude показывает написанный или сгенерированный код. Файл указан в хедере (`src/voice/rtc-pipeline.ts`).

### 4.8 Diff Blocks (блоки изменений)
**Компонент:** `DiffBlock`  
**Элементы:**
- Хедер: иконка редактирования, имя файла, счётчик `+42 -28` (зелёный/красный)
- Тело: строки с префиксами `-` (красные, удалённые) и `+` (зелёные, добавленные)

**Кейс:** Claude показывает diff изменений в существующем файле — что удалено, что добавлено.

### 4.9 Terminal Output (вывод терминала)
**Компонент:** блок с хедером Terminal + иконка терминала  
**Визуал:** тёмный фон кода, моноширинный шрифт  
**Кейс:** Результат выполнения bash-команды (тесты, сборка, lint). Пример: вывод `pnpm test` с результатами: `✓ 5 passed`, время `3.2s`.

### 4.10 Loop Progress (прогресс итерации)
**Компонент:** `LoopProgress`  
**Элементы:** спиннер, метка, прогресс-бар с плавной анимацией (`width transition`), счётчик `current/total`  
**Кейс:** Claude выполняет итеративный цикл — write-test-fix. Прогресс-бар показывает текущую итерацию из общего количества (`5/5`).

### 4.11 Inline Code
Внутри текстовых ответов — моноширинные `code` спаны с фоном `bg-elevated`. Используются для ссылок на переменные, параметры, команды.

### 4.12 Summary Stats (сводка результатов)
Строка в конце ответа с метриками: «3 files modified», «1 file created», `+42 lines`, `-28 lines`, «5/5 tests pass». Моноширинный шрифт, приглушённый цвет с зелёным/красным для additions/deletions.

### 4.13 Token Counter
**Компонент:** `TokenCounter`  
**Визуал:** строка рядом с именем Claude — `↑ 12.4k ↓ 3.2k Σ 15.6k tokens`  
**Кейс:** Позволяет отслеживать расход контекстного окна по каждому ответу.

### 4.14 Streaming Indicator
**Состояние:** когда Claude генерирует ответ  
**Элементы:**
- Метка «streaming» с анимированной иконкой refresh рядом с именем
- `TypingBar` — анимированная горизонтальная полоска акцентного цвета
- Текущий `ToolCallBlock` в состоянии `running`
- `ThinkingBlock` без контента (только пульсирующие точки)

**Кейс:** Пользователь видит, что Claude активно работает — reading, thinking, executing.

## 5. Approval Flow (запрос разрешений)

### 5.1 Permission Request
**Компонент:** `ApprovalBlock`  
**Визуал:** жёлтая левая граница, иконка alert, фон elevated  
**Элементы:**
- Заголовок «Permission Required»
- Команда в code-блоке: тип тула + команда (`Bash: git add -A && git commit -m "..."`)
- Три кнопки: **Allow** (акцент), **Deny** (серый), **Always allow** (приглушённый, справа)

**Кейс:** Claude хочет выполнить потенциально опасную операцию (commit, delete, install). Пользователь решает: разрешить однократно, запретить, или разрешить навсегда для этого типа операций.

## 6. MCP (Model Context Protocol)

### 6.1 Список серверов
Вкладка «MCP» в сайдбаре. Каждый сервер отображает: имя, количество доступных инструментов, статус подключения.

### 6.2 Статусы подключения
Три состояния с цветовой индикацией:
- **connected** — зелёная точка (filesystem, github, notion)
- **disconnected** — серая точка (postgres)
- **error** — красная точка (browser)

### 6.3 Управление серверами
Кнопка настроек у каждого сервера для конфигурации. Кнопка «Add MCP Server» внизу списка для подключения новых серверов.

---

## 7. Memory (память проекта)

### 7.1 Project Memory
Список правил и предпочтений, которые Claude запоминает по проекту:
- «Prefers TypeScript strict mode»
- «Uses pnpm as package manager»
- «Convention: kebab-case for file names»

Каждый элемент с кнопкой удаления (×).

### 7.2 Добавление воспоминаний
Кнопка «Add Memory» для ручного добавления правил.

### 7.3 CLAUDE.md
Отображение и редактирование файла `CLAUDE.md` — конфигурационного файла проекта, который Claude читает при старте сессии. Показывает содержимое (стек, API, голосовые технологии). Кнопка «Edit CLAUDE.md» для прямого редактирования.

---

## 8. Settings (настройки)

Слайд-панель справа с backdrop-blur оверлеем. Секции организованы как collapsible-блоки.

### 8.1 Model (выбор модели)
Три варианта: `claude-opus-4-0626`, `claude-sonnet-4-0514`, `claude-haiku-4-5`. Активная модель выделена акцентным фоном.

### 8.2 Permissions (разрешения)
Toggle-переключатели для пяти типов операций:
- File read (вкл)
- File write (вкл)
- Bash commands (вкл)
- Network access (выкл)
- MCP tools (выкл)

Анимированные переключатели с `cubic-bezier` transition.

### 8.3 Appearance (тема)
Переключатель Dark / Light с иконками луны и солнца. Смена темы анимируется через CSS transition на всех `var()` переменных (`0.4s ease`).

### 8.4 Loop Settings
Селектор максимального количества итераций: `5`, `10`, `25` (активен), `50`, `∞`.

### 8.5 Context (контекстное окно)
Прогресс-бар заполнения контекстного окна: `48.2k / 200k` tokens. Визуальная индикация — акцентный бар на сером фоне.

---

## 9. Навигация и Layout

### 9.1 Sidebar Toggle
Кнопка в top bar скрывает/показывает сайдбар. Анимация `fade-in` при появлении.

### 9.2 Sidebar Tabs
Три вкладки с иконками и метками: Sessions, MCP, Memory. Активная вкладка с акцентным фоном и цветом.

### 9.3 Responsive Chat Area
Чат ограничен `maxWidth: 820px` с центрированием. Скроллируемая область с авто-прокруткой к последнему сообщению.

### 9.4 Top Bar
Отображает: toggle сайдбара, название активной сессии, путь проекта (pill), статус соединения.

---

## 10. Темы

### 10.1 Dark Theme
Основной фон: `#282a36` (Dracula base). Все оттенки строятся от этого цвета с единым синевато-фиолетовым подтоном. Акцент: `#7c6cf0`.

### 10.2 Light Theme
Основной фон: `#fafafa`. Оттенки: серо-лавандовые (`#f0f0f3`, `#e8e8ee`, `#dddde6`). Акцент: `#6c5ce7`.

### 10.3 Переключение
Плавный transition `0.4s ease` на всех CSS-переменных. Кнопка в сайдбаре (sun/moon) или в настройках.

## 11. Анимации

| Анимация | Где используется | Описание |
|----------|-----------------|----------|
| `slide-up` | Сообщения, блоки инференса | Появление с offset 8px снизу + fade |
| `fade-in` | Сайдбар, file pills, drag overlay | Плавное появление opacity 0→1 |
| `spin` | ToolCall running, Loop spinner, Streaming | Непрерывное вращение 360° |
| `pulse-dot` | PulsingDots (thinking, streaming) | Масштабирование + opacity 3 точек со сдвигом |
| `typing-bar` | TypingBar (streaming) | Горизонтальная полоска пробегает слева-направо-обратно |
| `mic-pulse` | Кнопка микрофона (запись) | Пульсирующая тень вокруг кнопки |
| `cubic-bezier(0.4,0,0.2,1)` | Chevron rotation, toggles, progress bars | Плавные easing-кривые для UI-элементов |

## 13. Структура сообщений

### 13.1 User Message
- Аватар (инициал «M» в квадрате), имя «You», timestamp
- Текст сообщения
- Прикреплённые файлы (File Pills)

### 13.2 Claude Message
- Аватар (молния в акцентном квадрате), имя «Claude», Token Counter
- Последовательность блоков инференса в хронологическом порядке:
  1. ThinkingBlock (thinking)
  2. ThinkingBlock (reading)
  3. ThinkingBlock (planning)
  4. ToolCallBlock стек (Read, Search)
  5. Текстовый ответ
  6. CodeBlock
  7. DiffBlock
  8. ToolCallBlock стек (Write, Edit)
  9. ThinkingBlock (subagent)
  10. ToolCallBlock (Bash)
  11. Terminal Output
  12. LoopProgress
  13. Финальный текст с inline code
  14. Summary Stats

# 09 — Single Source of Truth (SSOT)

> Не допускаются двойные трактовки, дублирование данных, дублирование логики. Единый источник истины.

## Почему

Дублирование — корень рассинхронизации. Если одна и та же информация хранится в двух местах, рано или поздно они разойдутся. И тогда: какое из двух значений правильное? Никто не знает. Система теряет целостность.

SSOT — это не про "хранить мало данных". Это про то, что для каждого факта есть ровно одно каноническое место. Всё остальное — производные, кэши, проекции. Они вычисляются, не хранятся.

## Правила

1. **Один факт — одно место хранения**. Модель задана в settings → берётся из settings. Не из переменной, не из env, не из аргумента CLI
2. **Нет копий состояния**. Если компоненту нужен `model` — он читает из стора, не получает через props chain с копированием
3. **Derived data вычисляется**. `sessionCount`, `isStreaming`, `canSend` — computed, не stored
4. **Типы определяются один раз**. `StreamEvent` описан в `core/types/`, импортируется отовсюду. Нет параллельных определений
5. **Конфигурация в одном месте**. Не в .env + settings.json + hardcode + CLI args

## Тест на нарушение SSOT

Задай вопрос: **"Если я изменю это значение, сколько мест в коде нужно обновить?"**

- Ответ **"одно"** → SSOT соблюдён
- Ответ **"два или больше"** → нарушение SSOT

## Хорошо

```typescript
// Один источник для темы
// settings.theme → CSS-переменные → все компоненты
// Изменил settings.theme → всё обновилось автоматически

// Один источник для типов
// core/types/stream-event.ts экспортирует StreamEvent
// main.ts импортирует из core/types
// renderer импортирует из core/types
// Нет дублирования определений
```

```typescript
// Computed, не stored
const canSend = computed(() =>
  store.sdkStatus === 'ready' &&
  store.ui.streamingSessionId === null
);
// canSend всегда актуален, невозможно "забыть обновить"
```

## Плохо

```typescript
// Дублирование данных: model в трёх местах
const settings = { model: 'sonnet' };           // 1. в настройках
const claude = new Claude({ model: 'sonnet' }); // 2. в конструкторе (копия)
const [model, setModel] = useState('sonnet');    // 3. в UI state (ещё копия)
// Изменил settings.model — два других места устарели
```

```typescript
// Дублирование логики: форматирование даты в двух компонентах
// SessionsTab.tsx
const time = new Date(s.lastModified).toLocaleDateString();
// TopBar.tsx
const time = new Date(session.lastModified).toLocaleDateString();
// Логика идентична → должна быть в одной функции
```

```typescript
// Параллельные определения типов
// types.ts:     interface Session { id: string; name: string; }
// sidebar.ts:   interface SidebarSession { id: string; name: string; }
// Два типа для одного и того же → один устареет
```

## Ограничения

- **Запрещено** хранить одно и то же значение в двух местах (state + БД + UI — через подписку, не копирование)
- **Запрещено** определять один и тот же тип/интерфейс в нескольких файлах
- **Запрещено** дублировать бизнес-логику. Вынести в shared-функцию
- **Запрещено** хранить computed-значения. `isStreaming = streamingSessionId !== null` — вычисляется, не записывается
- **Обязательно** при обнаружении дубля — немедленный рефакторинг к единому источнику
